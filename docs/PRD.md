# PRD: 카셰어링 주행요금 계산기

## 목표
투루카·쏘카의 구간별 차등 요금제를 정확하게 계산해 주는 웹 계산기.  
사용자가 주행거리와 km당 단가를 입력하면 실제 청구될 주행요금과 구간별 내역을 즉시 확인할 수 있다.

---

## 사용자

### 페르소나
- **예약 전 확인자**: 투루카 또는 쏘카로 이동을 계획 중이며, 예약 전 주행요금이 얼마 나올지 미리 알고 싶다.
- **요금 비교자**: 이동 거리를 알고 있고, 투루카와 쏘카 중 어떤 서비스가 더 저렴한지 각각 계산해 비교하려 한다.

### Pain Point
- 쏘카·투루카 앱 내 요금 안내가 구간별로 나뉘어 있어 총 예상 요금을 직접 계산하기 번거롭다.
- 구간별 차등 요금 구조를 모르면 단순 km × 단가로 계산해 실제 청구 금액과 차이가 생긴다.

### 사용 시나리오
1. **예약 전 확인**: "이번 주말 부산 여행 왕복 300km면 투루카 주행요금이 얼마지?" → 거리 입력 → 즉시 확인
2. **쏘카 차종 비교**: 쏘카 앱에서 A 차종 단가를 확인 → 계산기에 입력 → 결과 확인 → B 차종 단가로 바꿔 재계산

## 요금 데이터 처리 방식

### 투루카
- 요금이 전국 동일·공개 고정값이므로 **코드에 하드코딩**한다.
- 요금 변경 시 `src/lib/turuka.ts`의 `TURUKA_TIERS` 상수만 수정.

### 쏘카
- **공개 API 없음** — 쏘카는 차종·쏘카존·시기에 따라 단가가 달라지며 외부에서 조회할 방법이 없다.
- 따라서 **사용자가 쏘카 앱에서 직접 단가를 확인하고 입력**한다.
- 사용자 흐름:
  1. 쏘카 앱에서 원하는 차종 선택
  2. 하단 '요금 안내' 탭에서 구간별 단가 확인
  3. 계산기에 단가 입력 → 거리 입력 → 결과 확인

### 2. 투루카 주행요금 계산

단가는 코드에 고정되어 있으므로 사용자는 거리만 입력한다.

#### 요금 구조
| 구간 | km당 요금 | 비고 |
|------|----------|------|
| 1 ~ 50km | 260원 | 경계값 50km 포함 |
| 51 ~ 100km | 250원 | 경계값 100km 포함 |
| 101km ~ | 230원 | |

출처: 투루카 공식 요금 안내

**구간 경계값 처리 규칙**
- 50km는 "1~50km" 구간(260원)에 포함된다.
- 51km부터 "51~100km" 구간(250원)이 시작된다.
- 100km는 "51~100km" 구간(250원)에 포함된다.
- 101km부터 "101km~" 구간(230원)이 시작된다.

#### 계산 방식
- 주행거리를 앞 구간부터 순서대로 채우며 각 구간 요금을 합산한다.
- 예: **80km** → (260원 × 50km) + (250원 × 30km) = 13,000 + 7,500 = **20,500원**
- 예: **120km** → (260원 × 50km) + (250원 × 50km) + (230원 × 20km) = **30,100원**
- 적용 거리가 0인 구간은 내역 행을 생성하지 않는다.

#### 입력
- 주행거리 (양의 정수 km, 최솟값 1)

#### 출력
- 구간별 요금 내역: 구간명 / 적용 km / km당 단가 / 소계
- 합계 금액 (천단위 콤마)

---

### 3. 쏘카 주행요금 계산

사용자가 쏘카 앱에서 확인한 단가를 직접 입력한다.  
차종 드롭다운 없음 — 단가 입력 방식이기 때문에 모든 차종·모든 쏘카존에 대응 가능하다.

#### 일반 차종 입력
사용자가 3개 구간의 단가를 각각 입력한다.

| 입력 항목 | 설명 |
|----------|------|
| 0~30km 단가 | 원/km, 쏘카 앱 요금 안내에서 확인 |
| 30~100km 단가 | 원/km |
| 100km~ 단가 | 원/km |
| 주행거리 | km |

#### 전기차 입력
"전기차" 토글을 켜면 단가 입력 필드가 1개로 줄어든다.

| 입력 항목 | 설명 |
|----------|------|
| 전 구간 단가 | 원/km (0이면 무료) |
| 주행거리 | km |

#### 계산 방식

**일반 차종**
- 투루카와 동일한 누적 합산 방식으로 3개 구간 계산
- 구간 경계값: 30km는 "0~30km" 구간, 31km부터 "30~100km" 구간
- 예: 123km, 단가 210/200/180원 → 6,300 + 14,000 + 4,140 = **24,440원**

**전기차**
- 전체 거리에 단일 단가 적용
- 예: 80km, 단가 150원 → **12,000원**

**무료 구간 처리**
- 단가 0원 구간은 소계를 "무료"로 표시한다.
- 전기차 단가 0원이면 합계 0원 + "무료" 표시.

#### 출력
- 구간별 요금 내역 (일반: 최대 3구간, 전기차: 단일 행)
- 합계 금액 (천단위 콤마)
- **전기차인 경우**: 탄소중립실천포인트 안내 배너
  - 문구: `최대 {주행거리 × 100}원 포인트 적립 가능 (연간 최대 70,000원)`

## 엣지 케이스

| 상황 | 처리 방식 |
|------|----------|
| 거리 미입력 (빈 값) | 결과 미표시, 힌트 문구 표시 |
| 거리 = 0 | 결과 미표시 |
| 거리 < 0 또는 문자 | `min="1"` + `inputmode="numeric"` 으로 차단 |
| 쏘카 단가 미입력 상태에서 거리 입력 | 결과 미표시, "단가를 먼저 입력해주세요" 안내 |
| 쏘카 단가 0원 입력 (의도적 무료 구간) | 해당 구간 소계 "무료" 표시 |
| 전기차 토글 켜고 단가 0원 | 합계 0원, "무료" 표시, 탄소중립 배너 유지 |
| 50km 정확히 입력 (투루카) | 1구간만 표시, 2구간 행 없음 |
| 30km 정확히 입력 (쏘카) | 1구간만 표시, 2구간 행 없음 |

## 결과 이후 사용자 경험
- 결과는 입력값 변경 즉시 자동 갱신된다. 별도 "계산" 버튼 없음.
- 거리 또는 단가를 수정하면 즉시 새 결과로 갱신된다.
- 전기차 토글 전환 시 단가 필드 초기화, 결과 초기화.
- 결과 복사·공유: MVP 제외.

- 시간요금, 보험료, 주차비, 반납 지연 요금 (주행요금만 계산)
